# Humanizer RU

[![Установки на skills.sh](https://skills.sh/b/smixs/humanizer-ru)](https://skills.sh/smixs/humanizer-ru)

Skill для AI-агентов: Claude Code, Codex, OpenClaw, Hermes и любых других, читающих формат SKILL.md. Удаляет признаки AI-генерации из русского текста: канцелярит, штампы ChatGPT и Claude, длинные тире, «не просто… а», рубленый драматизм, техножаргон. 34 паттерна, жёсткие запреты и защита от переусердствования.

Подходит для постов, статей, лендингов, документации и коммерческих текстов: гуманизация AI-текста, редактура по Ильяхову, чистка слопа и воды. Текст перестаёт палиться как сгенерированный и звучит как написанный человеком.

## Версия 1.5 - зачем обновили

Методология из [microsoft/SkillOpt](https://github.com/microsoft/SkillOpt), гейт - из [skill-conductor v3.1.0](https://github.com/smixs/skill-conductor). На 103 проверках счёт вырос с 98 до 100.

У агентов-редакторов есть предсказуемая болезнь: вместо одного слопа они пишут другой. «Не только X, но и Y» превращается в «как X, так и Y», «ключевой» - в «важнейший». Промптом это не лечится - в 1.4 проверку делает скрипт, а не сама модель.

**Добавлено:**

- **Работа в три фазы.** Сначала аудит с таблицей находок, потом точечная правка, потом проверка. Модель, которая правит по ходу чтения, пропускает половину паттернов
- **Линтер `scripts/lint.py`.** Жёсткие запреты проверяет греп, а не совесть модели: длинное тире, мат-знаки, негативные параллелизмы, рубленый драматизм. Ноль ошибок - текст готов. Питона нет - остаётся ручной чеклист
- **Паттерны 33 и 34.** Повтор глагола в соседних предложениях и абзацы-стопка без связок. В слепых тестах живые читатели ловят AI чаще всего на этих двух
- **Запрещённые замены.** Таблица «было / так нельзя / лечение»: удали, замени фактом или скажи проще. Синоним лечением не считается
- **Чистый текст остаётся чистым.** Нет паттернов - скилл возвращает оригинал без правок. Повторная редактура хорошего текста его портит
- **`knowledge/corrections.md`.** Журнал вашего фидбека: одно замечание превращается в постоянное правило
- **Справочник и эвалы.** SKILL.md стал короткой картой, 34 паттерна с примерами живут в `references/patterns.md`; в `evals/evals.json` 7 сценариев и 50 проверок

## Версия 1.2 - зачем обновили

Модели меняются. Каждое новое поколение пишет код всё лучше, а естественным языком - всё хуже: их затачивают под программирование, и эта заточка протекает в обычную речь. Модель объясняет отпуск как pull request: «скорость > красоты», «идея → прототип → продукт», «итерируемся по гипотезам». Человек так не говорит. Версия 1.2 бьёт именно по этому.

**Добавлено 11 паттернов (22-32):** стоп-слова и усилители, математические и кодовые знаки, техножаргон без пояснения, псевдоглубина, анонсы вместо дела, рубленый драматизм, формулы афоризмов, фальшивая доверительность, заголовок и его пересказ, резюмирующие заключения, спекулятивное заполнение пробелов.

**Изменено:**

- Правило «оценка меняется на факт»: «прочный» превращается в «выдерживает 150 кг». Свой вывод читатель запомнит, чужой - нет (из «Пиши, сокращай» Ильяхова)
- Уровень подачи: умный 12-летний подросток. Он поймёт любую мысль, если сказать по-человечески. Научный термин можно, но при первом появлении поясни одной фразой
- Проверка на уровне абзацев: главная мысль в начале; абзац не даёт читателю ничего нового - удаляется целиком
- Раздел «Что не трогать»: искать кластеры признаков, а не одиночные слова. Цитаты, странные конкретные детали, смешанные чувства и самопоправки - признаки живого человека, их сохраняем
- Шаг самопроверки: перед выдачей спросить себя «что в этом тексте всё ещё выдаёт AI» и переписать найденное
- «Режь воду, а не фактуру»: факты, цифры и примеры оригинала остаются

Версия сверена с полным текстом эссе [Википедия:Признаки сгенерированности текста](https://ru.wikipedia.org/wiki/Википедия:Признаки_сгенерированности_текста). Не переносили только вики-специфику: подлог источников и фейковые ISBN/DOI (это проверка фактов, а не стиля), Markdown в викитексте, несуществующие категории.

## Установка

Рекомендуемый способ - через [skills.sh](https://skills.sh/smixs/humanizer-ru), открытый маркетплейс агентских скиллов. Одна команда ставит skill в любой из 70+ поддерживаемых агентов: Claude Code, Codex, Cursor, OpenClaw, Hermes, Gemini CLI, OpenCode, Windsurf и другие.

```bash
npx skills add smixs/humanizer-ru
```

CLI сам определит установленных агентов и спросит, куда ставить. Полезные флаги:

```bash
# глобально, для всех проектов
npx skills add smixs/humanizer-ru -g

# в конкретные агенты
npx skills add smixs/humanizer-ru -a claude-code -a codex

# без вопросов, для CI
npx skills add smixs/humanizer-ru -g -y
```

Обновление до свежей версии: `npx skills update humanizer-ru`.

**Вручную (Claude Code):**

```bash
git clone https://github.com/smixs/humanizer-ru.git ~/.claude/skills/humanizer-ru
```

**Совсем вручную:** с версии 1.4 скилл многофайловый - карта `SKILL.md`, справочник `references/patterns.md`, линтер `scripts/lint.py`. Скопируйте все три в папку скиллов агента (`~/.claude/skills/`, `~/.codex/skills/`, `~/.openclaw/skills/`, `~/.hermes/skills/`):

```bash
mkdir -p ~/.claude/skills/humanizer-ru/{references,scripts}
base=https://raw.githubusercontent.com/smixs/humanizer-ru/main
curl -o ~/.claude/skills/humanizer-ru/SKILL.md $base/SKILL.md
curl -o ~/.claude/skills/humanizer-ru/references/patterns.md $base/references/patterns.md
curl -o ~/.claude/skills/humanizer-ru/scripts/lint.py $base/scripts/lint.py
```

Линтер - обычный python3 без зависимостей. Если питона нет, скилл продолжает работать по ручному чеклисту.

## 34 паттерна

**Канцелярит:** отглагольные существительные, цепочки родительных падежей, избегание «это»/«есть», пассивные конструкции, иностранные слова без нужды.

**AI-словарь и раздувание:** перегруженные слова, раздувание значимости и известности, поверхностный анализ с деепричастиями, рекламный язык, размытые атрибуции, шаблонные переходы, «вызовы и перспективы».

**Структурные:** негативные параллелизмы, правило трёх, синонимы-циклы, ложные диапазоны, рубленый драматизм, заголовок и его пересказ, резюмирующие заключения, повтор глагола в соседних предложениях, стопка абзацев без связок.

**Тон и коммуникация:** подобострастие, артефакты чатбота, хеджирование, пустые позитивные концовки, псевдоглубина, анонсы вместо дела, формулы афоризмов, фальшивая доверительность, спекулятивное заполнение пробелов.

**Лексика и знаки:** стоп-слова и усилители, математические и кодовые знаки, техножаргон без пояснения, эмодзи и избыточное форматирование.

## Жёсткие запреты

- «не просто X, а Y», «не только X, но и Y»
- длинное тире «—» (только короткое «-»)
- риторические вопросы
- математические и кодовые знаки в тексте: = → > < + vs &
- техножаргон без пояснения по-человечески
- стопки рубленых фрагментов подряд («Без X. Без Y. Только Z.»)
